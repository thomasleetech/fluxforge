CallCapture
Voice bot testing tool for QA teams — capture, transcribe, and analyze voice bot conversations at scale.
Enterprise Voice AI WebRTC
Idea to demo: 2 days

⚠ The Problem

QA teams working on enterprise voice bots had no systematic way to test conversations. They were manually calling their own bots, scribbling notes, and filing issues in spreadsheets. Every test was unrepeatable, unrecorded, and unstructured. They needed a tool that could initiate calls, capture transcripts in real time, and log issues directly against specific moments in the conversation.

✅ The Outcome

A fully functional call analysis platform with real-time transcription, conversation playback, and an issue tracking dashboard. QA teams could initiate test calls, watch transcripts appear live, flag problems inline, and export structured reports. Used by enterprise voice AI teams to systematically validate bot behavior across hundreds of scenarios.

Demonstrates FluxForge's ability to integrate complex real-time systems (WebRTC, speech-to-text, live UI updates) into a cohesive product in days, not months.

⚡ The Build

Movie Trailer Moment
Watching a live call transcript appear word-by-word in real time, synchronized with the actual conversation as it happens.
Vertical Slice
Initiate call → connect via WebRTC → stream audio to Whisper → display real-time transcript → log issues against transcript segments.
Stack
PHP backend, WebRTC for call handling, OpenAI Whisper for transcription, vanilla JavaScript frontend with live-updating DOM.

VibePad
A feature-rich code editor built entirely with vanilla JavaScript — no React, no Vue, no frameworks.
Developer Tools Vanilla JS Zero Frameworks
Idea to demo: 3 days

⚠ The Problem

The prevailing wisdom says you need React or Vue to build anything complex on the web. VibePad was born to disprove that. The challenge: build a fully featured, IDE-like code editor — syntax highlighting, live preview, theme engine, multi-file support — using nothing but vanilla JavaScript. No build tools, no bundlers, no framework tax.

✅ The Outcome

A complete code editor with syntax highlighting for multiple languages, a custom theme engine with dark and light modes, Canvas-powered minimap, code intelligence features, multi-file project support, and integrated live preview. All running as a single HTML file with zero dependencies.

VibePad is the definitive proof of the FluxForge "zero frameworks" principle at scale. It shows that vanilla JS isn't just viable for complex applications — it's often faster to build with, easier to debug, and lighter to ship.

⚡ The Build

Movie Trailer Moment
Opening the editor, pasting in code, and watching syntax highlighting + live preview render instantly — all powered by zero external libraries.
Vertical Slice
Text input → real-time syntax highlighting → live preview panel → file save/load → theme switching.
Stack
Pure vanilla JavaScript, Canvas API for minimap rendering, custom tokenizer for syntax highlighting, custom CSS-in-JS theme engine — all hand-rolled.

CV Pool
Play pool with your hands — computer vision-powered billiards using MediaPipe hand tracking.
Computer Vision Gaming MediaPipe
Idea to demo: 2 days

⚠ The Problem

Gesture-based gaming has always felt gimmicky. The Wii waggle, the Kinect dance — fun as novelties, but the controls never felt precise or responsive enough for a skill-based game. The question: can you make a gesture-controlled game that actually feels good to play? Can hand tracking be responsive and accurate enough for a game that requires fine motor control, like pool?

✅ The Outcome

A fully playable billiards game controlled entirely by hand gestures via a webcam. Smooth hand tracking maps your real hand position to a virtual cue stick. Realistic ball physics, pocket detection, and satisfying collision sounds. Projection-ready for use on real pool tables — turning any surface into an interactive game.

Demonstrates the "Feel Real" pillar in its purest form: responsive, tactile interaction that makes technology disappear behind the experience.

⚡ The Build

Movie Trailer Moment
Seeing your hand tracked on screen, pulling back a virtual cue stick, and watching the ball roll across the table with realistic physics.
Vertical Slice
Camera feed → MediaPipe hand detection → cue stick angle/power rendering → ball physics simulation → pocket detection → score tracking.
Stack
MediaPipe Hands for tracking, Canvas 2D for rendering, custom physics engine for ball collisions and momentum, vanilla JavaScript for game logic.

TimeVampire
AI-powered spam call interceptor that wastes scammers' time with convincing AI conversation.
Voice AI Consumer Multi-API
Idea to demo: 1 day

⚠ The Problem

Robocalls waste billions of hours of human time every year. Call blockers help, but scammers adapt. TimeVampire flips the script: instead of blocking the call, what if you answered it — with an AI that's specifically designed to waste the scammer's time? Keep them engaged, string them along, and make every minute they spend on your AI one less minute they spend scamming a real person.

✅ The Outcome

A working spam interceptor that picks up incoming calls, detects likely spam, and engages the caller with an AI personality. The AI generates natural conversational responses, asks clarifying questions, pretends to look for its "credit card," and generally keeps scammers on the line for as long as possible. Multiple AI personalities available for variety.

Demonstrates FluxForge's ability to rapidly integrate multiple AI APIs (speech-to-text, language model, text-to-speech, telephony) into a cohesive real-time system.

⚡ The Build

Movie Trailer Moment
A scammer calls in, the AI picks up with a friendly "Hello?", and a five-minute conversation unfolds where the scammer never realizes they're talking to a machine.
Vertical Slice
Incoming call via Twilio → AI picks up → speech-to-text transcription → OpenAI generates conversational response → ElevenLabs speaks the response → loop continues until scammer hangs up.
Stack
Twilio for telephony, OpenAI for conversation generation, ElevenLabs for realistic text-to-speech, PHP backend orchestrating the real-time loop.

CV Bowling
Gesture-controlled bowling with hand tracking — swing your arm and watch the pins fly.
Computer Vision Gaming Physics
Idea to demo: 2 days

⚠ The Problem

Can you make bowling feel physical using just a webcam? No special controllers, no motion sensors — just your arm, a camera, and a browser. The challenge is mapping the full arc of a bowling throw (approach, backswing, release, follow-through) to gesture data and translating that into satisfying pin-smashing physics.

✅ The Outcome

An arcade-style bowling game controlled entirely by arm gestures. The camera tracks your arm swing, detects the release point, calculates velocity and angle, and launches the ball down the lane. Realistic pin collision physics, lane animations, and satisfying sound design make every strike feel tangible and rewarding.

Proves that the "Feel Real" pillar extends beyond visual polish — sound, physics, and responsiveness combine to create a genuinely physical-feeling digital experience.

⚡ The Build

Movie Trailer Moment
Swinging your arm in front of the webcam and watching the bowling ball roll down the lane, curving with your wrist angle, and crashing into pins with full physics.
Vertical Slice
Camera feed → MediaPipe arm/hand gesture detection → release point calculation → ball trajectory with spin → pin collision physics → score display.
Stack
MediaPipe for gesture tracking, Canvas/WebGL for lane and pin rendering, custom physics simulation for pin collisions, Web Audio API for sound design.

InviteGate
Drop-in access control with invite codes — gate your beta without building full auth.
SaaS Security Access Control Reusable
Idea to demo: 3 hours

⚠ The Problem

You've built a prototype and you want to share it with a small group of beta testers. But you don't need a full auth system yet — no sign-up flows, no password resets, no OAuth integrations. You just need a gate: give someone a code, they get in. Simple, secure, and deployable in minutes. Every FluxForge project that reaches the "Prove Outcomes" phase needs this, and building it fresh each time is wasteful.

✅ The Outcome

A single-file, drop-in access control system. Generate invite codes, set usage limits and expiry dates, and gate any page or application behind a clean code-entry UI. Session-based access persists so users don't need to re-enter codes. Admin dashboard shows which codes are active, how many times each has been used, and when they expire.

Now reused across multiple FluxForge projects. Demonstrates the "Harden" pillar perfectly: simple security that works, deployed in minutes, and battle-tested across dozens of applications.

⚡ The Build

Movie Trailer Moment
Entering an invite code on a clean, minimal gate page and being instantly redirected to the protected application — the entire experience taking under two seconds.
Vertical Slice
Code input → server-side validation → session creation → redirect to protected content → admin code management dashboard.
Stack
PHP single-file deployment, MySQL for code storage and usage tracking, vanilla JS for the gate UI. No dependencies, no build step.
